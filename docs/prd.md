# Product Requirements Document (PRD): 遊戲市場成功率分析系統 (Game Success Analyzer)

## 1. 產品願景與目標 (Product Vision & Goals)
**願景**：提供遊戲開發者與投資人一個數據驅動的決策平台，透過分析市面上各類型遊戲的數據與成功因素，預測並產出製作特定類型遊戲的「獲利成功率」。
**目標**：
- 蒐集並聚合各大遊戲平台（如 Steam）的遊戲數據。
- 建立數據分析模型，找出影響遊戲成功（營收、玩家留存、評價）的關鍵因素。
- 提供使用者一個簡單的操作介面，輸入打算開發的遊戲類型與特徵，系統輸出預估的成功率及財務預期。
- **[V2]** 提供「獨立開發者友好型」的藍海市場偵察，識別低開發門檻、高回報的利基組合。

## 2. 目標受眾 (Target Audience)
- **獨立遊戲開發者 (Indie Developers)**：在立項前評估市場競爭度與潛在回報。
- **遊戲發行商與投資人 (Publishers & Investors)**：評估投資或代理某款遊戲原型的風險與勝率。
- **遊戲企劃 (Game Designers)**：尋找當前市場上的藍海市場或高轉換率的遊戲機制。

## 3. 使用者故事 (User Stories)
1. **作為系統管理員**，我希望系統能自動（或排程）從指定平台的 API / 網頁抓取遊戲歷史數據（包含類型、定價、銷量估算、玩家評論時間序列等），以便擁有持續更新的訓練與分析資料庫。
2. **作為使用者**，我希望能在首頁看到目前市場上最賺錢（成功率最高）的「Top 5 遊戲類型」趨勢儀表板，以便我快速掌握市場脈動。
3. **作為使用者**，我希望可以輸入我預計開發的遊戲參數（如：標籤、核心機制、目標平台、預計開發成本），系統能根據歷史數據模型，分析並產出「市面上同類產品的獲利成功率（如：75% 機率回本）」及一份簡要的「成功原因分析報告」。
4. **[V2] 作為獨立開發者**，我希望系統能自動「偵察」目前市場上的藍海市場，並根據「開發成本 (美術/程式/維運)」與「預估營收」的比值，推薦給我適合少量資源投入的「黃金開發配方」。

## 4. 驗收標準 (Acceptance Criteria - AC)
### AC 1: 數據來源與預處理模組 (MVP)
- 系統支援載入第三方遊戲資料集（例如匯入 Kaggle 的 Steam Games CSV 檔案）。
- 資料處理邏輯需清理資料並寫入資料庫，欄位包含：遊戲名稱、標籤(類型)、價格、發行時間、預估擁有者數量 (Estimated Owners)。

### AC 2: 市場趨勢儀表板 (Dashboard)
- 前端首頁必須展示包含圖表的儀表板。
- 系統將「成功」這件事量化：例如定義「成功 = (價格 * 預估擁有者數量) > 10,000 美金 (可配置參數)」。
- 圖表需正確渲染各標籤/類型遊戲的「成功佔比」或「中位數營收」排行。

### AC 3: 成功率預測分析工具 (Predictor)
- 系統需提供一個表單，讓使用者選擇「遊戲類型 / 標籤」並填寫基本參數。
- 送出表單後，後端需執行分析演算法，回傳介於 0%~100% 的成功預測率，並附帶文字說明。

### AC 4: [V2] 藍海市場偵察報告 (Blue Ocean Recon)
- **分析引擎**：實作「資源成本三維係數 (Complexity Triple-Index)」：
    - **美術投入**：識別像素 (Pixel)、2D、極簡 (Minimalist) 等標籤。
    - **軟體難度**：評估多人連線、複雜物理等技術門檻。
    - **維運難度**：以「內容消耗率」為核心指標，分析玩家重玩價值。
- **偵察按鈕**：首頁需提供「🔍 偵察市場藍海」入口。
- **策略報告**：點擊後以精美 Modal 展示 Top 3 推薦開發配方，包含配方名稱、標籤組合、三維成本圖形化標記（🟢🟡🔴）、分析師短評與風險提示。

## 5. 里程碑與範圍限制 (Out of Scope for V1/V2)
- V1/V2 暫不實作複雜的機器學習預測模型，採用基於歷史數據的「過濾與加權平均法」進行統計與評分。
- V2 導入 AI 專業叙事邏輯，用於生成報告中的「分析師短評」。
- 本專案限定為 Web 應用程式介面。
