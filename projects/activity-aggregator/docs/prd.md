# 產品需求文件 (PRD) - 萬能活動整理系統 (Activity Aggregator)

## 1. 願景與目標 (Vision & Objectives)
*   **目標**：建立一個自動化的活動彙整平台，主動從各個分散的平台（如：在地活動網、旅遊平台、社群媒體）採集資訊。
*   **核心價值**：解決「資訊碎片化」的問題。使用者不需要登入多個 App，系統會根據其偏好，自動統整適合的活動供選擇。

## 2. 目標用戶 (Target Audience)
*   希望節省搜尋時間，尋找週末或空閒時間高品質活動的使用者。

## 3. 使用者故事 (User Stories) & 驗收標準 (AC)

### [P0] US1: 偏好設定 (User Preference Profile)
> 身為一個使用者，我想要設定我的活動偏好（如：地區、預算、興趣標籤），以便系統精準統整。
*   **AC1**: 提供標籤選擇介面（例如：運動、藝術、美食、大自然）。
*   **AC2**: 支援設定地理位置範圍（例如：台北市、捷運沿線）。

### [P0] US2: 自動彙整清單 (Multi-platform Aggregation)
> 身為一個使用者，我想要能看到從不同平台統整後的活動列表。
*   **AC1**: 列表需標示活動來源（例如：來自 Klook, Accupass 或在地政府官網）。
*   **AC2**: 列表應自動過濾掉不符合使用者偏好的活動。
*   **AC3**: 活動資訊需包含包含：名稱、日期/時間、地點、費用、原始連結。

### [P1] US3: 智慧推薦與去重 (Smart Recommendation & Deduplication)
> 身為一個使用者，我不想要看到重複的活動，且希望看到最適合我的推薦。
*   **AC1**: 系統需具備基本的去重邏輯，若多平台有相同活動應只顯示一筆。
*   **AC2**: 推薦演算法需根據標籤權重排序顯示順序。

### [P1] US4: 快速導流 (Direct Routing)
> 身為一個使用者，當我選定活動時，我想要能直接進入來源網頁進行訂盤或查看詳情。
*   **AC1**: 提供「前往原站點」按鈕。
*   **AC2**: 確保導引連結的準確性。

## 4. 平台整合策略 (Data Sourcing Strategy)
*   **初期 (MVP)**：整合 2-3 個基礎 API 或 靜態來源（如：Open Data 活動介面）。
*   **後期**：導入 Web Scraper 或 AI 驅動的動態頁面解析。

## 5. MVP 範圍 (MVP Scope)
1.  使用者偏好設定（標籤化選擇）。
2.  基礎 2 個來源平台的活動資料讀取與展示。
3.  列表篩選與跳轉連結。
